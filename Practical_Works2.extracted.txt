PROJECT 2: AUTHENTICATION SYSTEM
PROJECT CONTEXT
Current Problems.
Students share passwords
No way to prove who actually accessed a system
No MFA for sensitive systems (grades, payroll)
Can't track login history for security audits
Password reset requires IT intervention
Objective: Implement a robust authentication system that verifies users' identity through multiple methods before granting access to university resources.

ACCEPTANCE CRITERIA
Level | Name | Methods
L1 | Basic | Password only
L2 | Standard | Password + SMS OTP
L3 | High | Password + SMS OTP + TOTP
L4 | Critical | Password + SMS OTP + TOTP + Secret Question.
User Type | Default Level | Can Upgrade To | Special Cases
Students | L1 (Basic) | L2 (optional) | International students: L2 mandatory
Faculty | L2 (Standard) | L3 (optional) | Department heads: L3 mandatory
Admin Staff | L2 (Standard) | L3 (optional) | HR/payroll staff: L3 mandatory
IT Admins | L4 (Critical) | N/A | Always L4
Researchers | L2 (Standard) | L3 (if grant-funded) | Lab access requires biometric
Contractors | L1 (Basic) | L2 (if approved) | Temporary, expires automatically
Length: Minimum 8 characters, maximum 64
Complexity: Must contain 3 of 4 categories
Uppercase letters (A-Z)
Lowercase letters (a-z)
Numbers (0-9)
Special characters (!@#$%^&*)
History: Cannot use last 5 passwords
Failed attempts: Lock after 5 failures for 30 minutes
First login: Must change password immediately
Personal info: Cannot contain name, username, birthdate
Hash algorithm: bcrypt with cost factor 12
Salt: Unique per password
TOTP (Time-based One-Time Password)

Algorithm: HMAC-SHA1
Time step: 60 seconds
Code length: 6 digits
Setup: QR code provisioning
Recovery: 10 backup codes (one-time use)
SMS/Email OTP

Code length: 8 digits
Validity: 5 minutes
Rate limit: Maximum 3 requests per hour
Resend: Cooldown 60 seconds
Invalid attempts: Block after 3 failures
Template: "Your university verification code is: {code}. Valid for 5 minutes."
Security Questions

Questions: User selects 2 from pool of 3
Answers: Case-insensitive hashed
Storage: bcrypt hashed, not encrypted
Use: Password reset OR second factor
Attempts: 3 max, then lockout
Sample questions
    What was your first pet's name?
    In which city were you born?
    What was your childhood nickname?
